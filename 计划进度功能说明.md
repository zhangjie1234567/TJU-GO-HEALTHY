# 计划进度功能说明

## 功能概述
实现了动态人物形象、实时数据统计和体重趋势图的完整计划进度功能。

## 核心改动

### 1. 数据服务层 (userProgressService.js)
**位置**: `src/utils/userProgressService.js`

**提供的功能**:
- **天数统计**: 使用天数、记录天数、坚持方案天数、剩余预估天数
- **体重数据**: 获取体重历史、当前体重、目标体重
- **BMI计算**: 计算BMI值并评估健康状态
- **人物形象系统**: 根据BMI状态返回对应的emoji
  - 🧘 偏瘦 (BMI < 18.5)
  - 📈 正常 (BMI 18.5-24)
  - 😊 偏胖 (BMI 24-28)
  - 😅 肥胖 (BMI ≥ 28)
- **智能鼓励语**: 根据用户进度生成动态鼓励文字

**核心函数**:
```javascript
// 获取完整用户进度数据
getUserProgressData()
// 返回11个字段的进度对象

// 获取动态鼓励语
getEncouragementText(progressData)
```

### 2. 计划进度页面 (plan_progress.vue)
**位置**: `src/pages/home/plan_progress.vue`

**主要功能**:
1. **动态人物形象区域**
   - 大号emoji显示（140px，随体重变化）
   - BMI徽章显示当前BMI值
   - 健康状态描述文字

2. **4种天数统计**（2x2网格布局）
   - 📅 使用天数：从首次使用到今天
   - ✍️ 记录天数：有任何记录的天数
   - 💪 坚持方案：完成餐食方案的天数
   - 🎯 剩余天数：预估达成目标的天数

3. **智能进度提示**
   - 根据进度百分比和剩余天数动态显示提示语
   - 提示卡片采用渐变背景

4. **体重趋势图**
   - 使用ECharts绘制平滑曲线
   - 渐变填充区域
   - 显示当前体重和目标体重
   - 目标体重虚线标记
   - 数据不足时显示友好提示（0数据/1数据）
   - 点击"去记录体重"按钮跳转到体重记录页面

5. **动态鼓励语**
   - 从userProgressService获取个性化鼓励语
   - 渐变背景卡片显示

**技术实现**:
- Vue3 Composition API
- ECharts图表库（UniApp适配版）
- 响应式数据绑定
- Canvas渲染优化

### 3. 首页人物形象卡片 (home.vue)
**位置**: `src/pages/home/home.vue`

**改动内容**:
- 替换静态图片为动态emoji显示
- 显示健康状态描述文字
- 右上角显示BMI徽章
- 添加浮动动画效果（3秒循环上下浮动）
- 点击跳转到计划进度页面

**样式特点**:
- 100px大小的emoji
- BMI徽章采用渐变蓝色背景
- 流畅的动画过渡
- 保持卡片48%宽度和280px高度

## 数据依赖

### LocalStorage存储键
| 键名 | 说明 | 格式 |
|------|------|------|
| weightList | 体重历史记录 | `[{date: 'YYYY-MM-DD', weight: number}]` |
| targetWeight | 目标体重 | `number` |
| calories | 每日餐食记录 | `{date: {breakfast, lunch, dinner, other}}` |
| daily_records | 每日记录标记 | `{date: true}` |
| firstUseDate | 首次使用日期 | `'YYYY-MM-DD'` |
| questionnaireBaseInfo | 问卷基础信息 | `{height: number, weight: number}` |

### 数据流
```
LocalStorage
    ↓
userProgressService.js (数据计算和聚合)
    ↓
plan_progress.vue / home.vue (UI展示)
```

## BMI计算标准（中国成人标准）
- **偏瘦**: BMI < 18.5 → 🧘
- **正常**: 18.5 ≤ BMI < 24 → 📈
- **偏胖**: 24 ≤ BMI < 28 → 😊
- **肥胖**: BMI ≥ 28 → 😅

## 剩余天数算法
```javascript
// 基于历史减重速度预估
平均每日减重 = (首次体重 - 最新体重) / 记录天数
剩余天数 = (当前体重 - 目标体重) / 平均每日减重

// 无历史数据时使用默认速度0.1kg/天
```

## UI优化亮点
1. **渐变背景**: 多处使用蓝色系渐变，提升视觉层次
2. **卡片阴影**: 统一使用淡蓝色阴影，增强立体感
3. **emoji动画**: 首页人物形象添加浮动动画，增加趣味性
4. **空状态友好**: 数据不足时显示引导提示和操作按钮
5. **网格布局**: 4种天数统计采用2x2网格，信息密度合理
6. **智能提示**: 根据完成度动态调整鼓励语和提示内容

## 测试建议
1. **无数据场景**: 清空体重记录，查看空状态提示
2. **单条数据**: 记录1次体重，验证"数据不足"提示
3. **多条数据**: 记录≥2次体重，查看趋势图渲染
4. **BMI变化**: 记录不同体重值，观察emoji和状态变化
5. **天数统计**: 添加餐食记录和日常记录，验证天数准确性

## 注意事项
1. **首次使用**: 需要先填写问卷获取身高体重数据
2. **趋势图渲染**: 需要至少2条体重记录才能显示
3. **BMI计算**: 依赖身高和体重数据，缺失时无法计算
4. **剩余天数**: 需要有减重趋势（体重下降）才能准确预估
5. **Canvas性能**: ECharts图表在低端设备可能有渲染延迟
