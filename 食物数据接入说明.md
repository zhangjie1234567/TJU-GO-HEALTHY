# 食物数据接入说明

## 📌 概述

本项目已实现完整的食物搜索和详情展示功能，当前使用模拟数据，预留了对接开源食物数据库的标准接口。

## 🗄️ 推荐的开源数据源

### 1. 中国食物成分表（推荐）
- **数据来源**: 中国疾控中心营养与健康所
- **数据格式**: Excel/数据库
- **获取方式**: 
  - 官方网站：http://www.chinanutri.cn
  - GitHub开源项目：搜索 "中国食物成分表"
  - 数据特点：覆盖2000+种中国常见食物，营养成分完整

### 2. USDA FoodData Central
- **数据来源**: 美国农业部
- **API接口**: https://fdc.nal.usda.gov/api-guide.html
- **数据特点**: 全球食物数据库，含50万+食物条目
- **使用方式**: 申请免费API Key即可使用

### 3. 薄荷健康食物库
- **数据来源**: 薄荷健康App
- **获取方式**: 可通过爬虫采集（注意版权）
- **数据特点**: 符合国人饮食习惯

## 🔌 对接真实数据库的步骤

### 方案A：直接API对接（推荐用于USDA等有API的数据源）

1. **修改 `foodDataService.js`**

```javascript
// 配置API基础信息
const API_BASE_URL = 'https://api.nal.usda.gov/fdc/v1'
const API_KEY = 'YOUR_API_KEY'

// 修改 searchFoods 函数
export async function searchFoods(keyword = '', category = '') {
  try {
    const response = await uni.request({
      url: `${API_BASE_URL}/foods/search`,
      method: 'GET',
      data: {
        api_key: API_KEY,
        query: keyword,
        pageSize: 20
      }
    })
    
    // 将API返回数据转换为我们的标准格式
    return response.data.foods.map(food => ({
      id: food.fdcId,
      name: food.description,
      category: mapCategory(food.foodCategory),
      energy: food.foodNutrients.find(n => n.nutrientId === 1008)?.value || 0,
      // ... 其他字段映射
    }))
  } catch (error) {
    console.error('API请求失败', error)
    return []
  }
}
```

### 方案B：本地数据库对接（推荐用于中国食物成分表）

1. **准备本地数据文件**
   - 将Excel数据转为JSON格式
   - 存放在 `static/data/foods.json`

2. **修改数据加载方式**

```javascript
// 在 foodDataService.js 中
let foodDatabase = []

// 初始化：加载本地数据
export async function initFoodData() {
  try {
    const response = await uni.request({
      url: '/static/data/foods.json',
      method: 'GET'
    })
    foodDatabase = response.data
    console.log('食物数据加载成功', foodDatabase.length)
  } catch (error) {
    console.error('数据加载失败', error)
  }
}

// 在 home_search_bar.vue 的 onMounted 中调用
import { initFoodData } from '../../utils/foodDataService.js'
onMounted(async () => {
  await initFoodData()
  // 其他初始化...
})
```

### 方案C：后端API对接（推荐用于生产环境）

1. **创建后端API服务**
   - 将食物数据导入MySQL/PostgreSQL
   - 使用Node.js/Python搭建RESTful API

2. **修改服务层**

```javascript
// 配置后端API地址
const API_BASE_URL = 'https://your-backend-api.com/api'

export async function searchFoods(keyword, category) {
  const response = await uni.request({
    url: `${API_BASE_URL}/foods/search`,
    method: 'GET',
    data: { keyword, category }
  })
  return response.data.data
}
```

## ⚡ 性能优化建议

1. **数据缓存**
```javascript
// 使用 uni.setStorageSync 缓存常用数据
export async function getFoodDetail(foodId) {
  const cacheKey = `food_${foodId}`
  const cached = uni.getStorageSync(cacheKey)
  
  if (cached && Date.now() - cached.timestamp < 86400000) {
    return cached.data
  }
  
  const data = await fetchFromAPI(foodId)
  uni.setStorageSync(cacheKey, {
    data,
    timestamp: Date.now()
  })
  return data
}
```

## 🆘 常见问题

**Q: 如何快速测试真实API？**
A: 先在 `foodDataService.js` 中只替换一个函数（如 `searchFoods`），保留其他模拟函数，逐步迁移。

**Q: 图片加载太慢怎么办？**
A: 使用CDN加速，或者将图片存储到云存储（阿里云OSS、腾讯云COS）。

**Q: 数据量大如何优化？**
A: 使用虚拟列表组件、分页加载、索引搜索。
