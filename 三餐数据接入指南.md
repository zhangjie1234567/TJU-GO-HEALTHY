# 开源食物数据接入指南

## 📚 概述

本文档介绍如何将开源食物数据库接入到 TJU-GO-HEALTHY 项目中，替换当前的模拟数据，实现真实的食物营养成分查询功能。

## 🌐 推荐的开源食物数据库

### 1. 中国食物成分表（Chinese Food Composition Tables）

#### 数据来源
- **官方机构**: 中国疾病预防控制中心营养与健康所
- **数据规模**: 包含2000+种中国常见食物的营养成分数据
- **数据项**: 能量、蛋白质、脂肪、碳水化合物、膳食纤维、维生素、矿物质等30+项营养成分

#### 获取方式
1. **官方数据库**: 
   - 网址: http://www.chinanutri.cn/
   - 可购买标准版食物成分表书籍或授权使用数据

2. **开源API服务**:
   - [FoodData Central API](https://fdc.nal.usda.gov/api-guide.html) (美国农业部)
   - [Open Food Facts API](https://world.openfoodfacts.org/data) (全球食品数据库)
   - [薄荷健康API](https://www.boohee.com/) (需要申请合作)

### 2. USDA FoodData Central

#### 特点
- **数据量大**: 超过45万种食物
- **数据详细**: 包含完整的营养成分、品牌信息等
- **免费API**: 提供RESTful API接口
- **更新频繁**: 定期更新数据

#### API使用示例
```javascript
// 获取API密钥: https://fdc.nal.usda.gov/api-key-signup.html
const API_KEY = 'YOUR_API_KEY'
const API_BASE = 'https://api.nal.usda.gov/fdc/v1'

// 搜索食物
async function searchFood(query) {
  const response = await fetch(
    `${API_BASE}/foods/search?query=${query}&api_key=${API_KEY}`,
    { method: 'GET' }
  )
  return await response.json()
}

// 获取食物详情
async function getFoodDetail(fdcId) {
  const response = await fetch(
    `${API_BASE}/food/${fdcId}?api_key=${API_KEY}`,
    { method: 'GET' }
  )
  return await response.json()
}
```

### 3. Open Food Facts

#### 特点
- **开源免费**: 完全开源，无需付费
- **全球化**: 包含全球食品数据
- **图片丰富**: 包含食物图片
- **社区驱动**: 由全球用户贡献数据

#### API使用示例
```javascript
const OFF_API = 'https://world.openfoodfacts.org/api/v2'

async function getProduct(barcode) {
```

## 🔧 接入实施方案

### 方案一：直接调用第三方API（推荐用于快速开发）

#### 优点
- 实施简单快速
- 数据实时更新
- 无需维护数据库

#### 缺点
- 依赖网络连接
- 受API限速限制
- 数据格式需要转换

#### 实施步骤

1. **修改 foodDataService.js**

```javascript
// src/pages/home/foodDataService.js

// 配置API
const FOOD_API_CONFIG = {
  provider: 'USDA', // 'USDA' | 'OpenFoodFacts' | 'Custom'
  apiKey: 'YOUR_API_KEY',
  baseUrl: 'https://api.nal.usda.gov/fdc/v1'
}

/**
 * 搜索食物（对接真实API）
 */
export async function searchFoods(keyword = '', category = '') {
  try {
    // 调用USDA API
    const response = await uni.request({
      url: `${FOOD_API_CONFIG.baseUrl}/foods/search`,
      method: 'GET',
      data: {
        query: keyword,
        api_key: FOOD_API_CONFIG.apiKey,
        pageSize: 20
      }
    })
    
    if (response.statusCode === 200 && response.data.foods) {
      // 转换API数据格式为项目标准格式
      return response.data.foods.map(transformApiFood)
    }
    
    return []
  } catch (error) {
    console.error('API请求失败:', error)
    // 降级到本地模拟数据
    return mockFoodDatabase.filter(item => 
      item.name.includes(keyword) && 
      (!category || category === '全部' || item.category === category)
    )
  }
}

/**
 * 转换API数据格式
 */
function transformApiFood(apiFood) {
  return {
    id: `usda_${apiFood.fdcId}`,
    name: apiFood.description,
    category: inferCategory(apiFood),
    image: getImageUrl(apiFood),
    energy: extractNutrient(apiFood, 'Energy'),
    nutrition: {
      water: extractNutrient(apiFood, 'Water'),
      protein: extractNutrient(apiFood, 'Protein'),
      fat: extractNutrient(apiFood, 'Total lipid (fat)'),
      cho: extractNutrient(apiFood, 'CHO'),
      dietaryFiber: extractNutrient(apiFood, 'Dietary fiber'),
      vitaminA: extractNutrient(apiFood, 'Vitamin A'),
      thiamin: extractNutrient(apiFood, 'Thiamin'),
      riboflavin: extractNutrient(apiFood, 'Riboflavin'),
      niacin: extractNutrient(apiFood, 'Niacin'),
      vitaminC: extractNutrient(apiFood, 'Vitamin C'),
      vitaminE: extractNutrient(apiFood, 'Vitamin E'),
      ca: extractNutrient(apiFood, 'Ca'),
      fe: extractNutrient(apiFood, 'Fe'),
      na: extractNutrient(apiFood, 'Na'),
      cholesterol: extractNutrient(apiFood, 'Cholesterol')
    },
    effect: '具有营养价值',
    feature: apiFood.additionalDescriptions || '营养丰富',
    relatedRecipes: [],
    tags: generateTags(apiFood)
  }
}

/**
 * 从API数据中提取特定营养素
 */
function extractNutrient(food, nutrientName) {
  if (!food.foodNutrients) return 0
  
  const nutrient = food.foodNutrients.find(n => 
    n.nutrientName && n.nutrientName.includes(nutrientName)
  )
  
  return nutrient ? nutrient.value : 0
}

/**
 * 推断食物分类
 */
function inferCategory(food) {
  const description = food.description.toLowerCase()
  
  if (description.includes('apple') || description.includes('banana') || 
      description.includes('orange')) {
    return '水果'
  }
  if (description.includes('vegetable') || description.includes('carrot')) {
    return '蔬菜'
  }
  if (description.includes('meat') || description.includes('beef')) {
    return '肉类'
  }
  if (description.includes('grain') || description.includes('rice')) {
    return '谷物'
  }
  
  return '其他'
}

/**
 * 获取食物图片URL
 */
function getImageUrl(food) {
  // 使用Unsplash的食物图片API作为默认
  return `https://source.unsplash.com/400x400/?${encodeURIComponent(food.description)},food`
}

/**
 * 生成标签
 */
function generateTags(food) {
  const tags = []
  
  // 根据营养成分生成标签
  const protein = extractNutrient(food, 'Protein')
  const fat = extractNutrient(food, 'Total lipid (fat)')
  const energy = extractNutrient(food, 'Energy')
  
  if (protein > 15) tags.push('高蛋白')
  if (fat < 3) tags.push('低脂肪')
  if (energy < 100) tags.push('低热量')
  
  return tags
}
```

### 方案二：本地数据库方案（推荐用于生产环境）

#### 优点
- 响应速度快
- 不受网络限制
- 可自定义完善数据
- 无API调用限制

#### 缺点
- 需要初始数据导入
- 需要定期更新
- 占用存储空间

#### 实施步骤

1. **获取食物数据CSV/JSON文件**
   - 从USDA下载FoodData Central数据集
   - 或使用中国食物成分表数据

2. **数据清洗与转换**

```javascript
// tools/data-import/convertFoodData.js
const fs = require('fs')
const csv = require('csv-parser')

async function convertFoodData(csvPath, outputPath) {
  const foods = []
  
  fs.createReadStream(csvPath)
    .pipe(csv())
    .on('data', (row) => {
      foods.push({
        id: `food_${row.fdc_id}`,
        name: row.description,
        category: inferCategory(row),
        energy: parseFloat(row.energy_kcal) || 0,
        nutrition: {
          protein: parseFloat(row.protein) || 0,
          fat: parseFloat(row.fat) || 0,
          cho: parseFloat(row.cho) || 0,
          // ... 其他营养成分
        }
      })
    })
    .on('end', () => {
      fs.writeFileSync(outputPath, JSON.stringify(foods, null, 2))
      console.log(`已导入 ${foods.length} 项食物数据`)
    })
}

convertFoodData('./usda_data.csv', '../src/data/foodDatabase.json')
```

3. **在项目中使用本地数据**

```javascript
// src/pages/home/foodDataService.js
import foodDatabase from '@/data/foodDatabase.json'

export async function searchFoods(keyword = '', category = '') {
  // 模拟异步延迟
  await new Promise(resolve => setTimeout(resolve, 300))
  
  let results = foodDatabase
  
  // 分类筛选
  if (category && category !== '全部') {
    results = results.filter(item => item.category === category)
  }
  
  // 关键词搜索
  if (keyword.trim()) {
    const lowerKeyword = keyword.toLowerCase().trim()
    results = results.filter(item => 
      item.name.toLowerCase().includes(lowerKeyword) ||
      item.tags.some(tag => tag.toLowerCase().includes(lowerKeyword))
    )
  }
  
  return results
}
```

### 方案三：混合方案（最优方案）

#### 架构设计
```
┌─────────────────┐
│   前端应用层    │
└────────┬────────┘
         │
         ↓
┌─────────────────┐
│  数据服务层     │ ← foodDataService.js
└────────┬────────┘
         │
    ┌────┴────┐
    ↓         ↓
┌────────┐ ┌────────┐
│ 本地缓存│ │ API层  │
└────────┘ └───┬────┘
                │
            ┌───┴───┐
            ↓       ↓
        ┌─────┐ ┌─────┐
        │USDA │ │开源 │
        │ API │ │数据 │
        └─────┘ └─────┘
```

#### 实现逻辑
```javascript
// 混合数据源策略
export async function searchFoods(keyword = '', category = '') {
  // 1. 优先从本地缓存查找
  const cachedResults = searchFromLocalCache(keyword, category)
  if (cachedResults.length > 0) {
    return cachedResults
  }
  
  // 2. 本地数据库搜索
  const localResults = searchFromLocalDB(keyword, category)
  if (localResults.length >= 5) {
    // 本地有足够数据，直接返回
    cacheResults(keyword, category, localResults)
    return localResults
  }
  
  // 3. 调用远程API补充
  try {
    const apiResults = await searchFromAPI(keyword, category)
    const combinedResults = [...localResults, ...apiResults]
    cacheResults(keyword, category, combinedResults)
    return combinedResults
  } catch (error) {
    // API失败，返回本地结果
    return localResults
  }
}
```

## ⚠️ 注意事项

### 法律合规
1. **数据版权**: 确保使用的数据有合法授权
2. **API条款**: 遵守API使用条款和限速规则
3. **隐私保护**: 不采集用户个人饮食数据用于其他用途

### 数据质量
1. **数据校验**: 对导入的数据进行质量检查
2. **单位统一**: 确保所有营养成分单位统一（如：每100g）
3. **缺失处理**: 对缺失的营养数据使用默认值或标记

### 性能优化
1. **分页加载**: 搜索结果分页返回
2. **图片优化**: 使用CDN加速图片加载
3. **缓存策略**: 实现多级缓存（内存+本地存储）
